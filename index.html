🛡
Input Validation
  • Mandatory fields enforced (invoice_id, vendor_id, date, items)
  • Invoice ID format: INV-XXXXX (regex validated)
  • GST number format: 15-char alphanumeric
  • Date range: not future-dated, not >90 days old
  • Subtotal + GST = Total (math check)
  • HSN code present and 4–8 digits
  • Blank quantity or zero-price lines → auto-reject
  • Negative line items → flag for review
⚖
Automated Reconciliation
  • Unit price vs contracted rate (tolerance: ±0.5%)
  • GST rate vs HSN-mandated rate from rate card
  • Billed amount = quantity × unit_price (line math)
  • Duplicate line detection (same item + same invoice)
  • Item name fuzzy-match against approved catalog
  • Unapproved service codes → flag immediately
  • Quantity spike >200% of 3-month avg → escalate
  • New item not in rate card → hold for approval
🎯
Exception Dashboard
  • Exceptions auto-routed by risk tier
  • One-click approve / reject / hold with audit trail
  • SLA timer: High → 24h, Medium → 72h, Low → 7 days
  • Escalation to CFO if unresolved past SLA
  • Vendor notified automatically on rejection
  • Credit note request generated on overcharge confirm
  • Human review required for >₹5L discrepancy
  • Payment blocked until exception is resolved

🔢 Math Check
billed_amount must equal qty × unit_price within ₹0.01. Invoice total must equal sum of line items + GST.
📋 Rate Card Match
Unit price automatically compared to contracted rate. Variance >0.5% triggers immediate flag.
🏷 GST Verification
GST rate cross-checked against HSN code. Wrong rate → invoice rejected. Must resubmit with correct rate.
🔁 Duplicate Detection
System checks same item + same vendor within 30-day window. Duplicates auto-flagged for consolidation.
📅 Date Validation
Invoice date cannot be in the future or more than 90 days old. Service date must precede invoice date.
📂 Catalog Check
Item name fuzzy-matched against approved catalog. Unrecognized items held for manual approval before processing.
